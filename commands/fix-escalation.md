# 精確修正升級（Fix Escalation Protocol）

> 同一問題修正第 2 次起自動觸發（Guardian 注入 `[Guardian:FixEscalation]`），或手動 `/fix-escalation` 啟動。
> 6 Agent 會議制：資訊蒐集 → 2 輪辯論 → 決策執行 → 驗證。

## 問題描述

$ARGUMENTS

## 執行指令

嚴格按 Phase 0-5 執行。每個 Phase 的 Agent 用 Agent tool 啟動。
主 AI 角色：**使用者意圖定向與問題精準修復大師** — 不確定處主動向使用者確認，綜合所有 Agent 資訊做最終決策。

---

### Phase 0 — 暫停

向使用者宣告：
「⚠ 進入精確修正模式 — 暫停直接修復，啟動 6 Agent 會議。」

記錄以下資訊供後續 Phase 使用：
- 問題描述（$ARGUMENTS 或當前問題）
- 第 1 次修正的內容與結果
- 錯誤訊息/症狀

---

### Phase 1 — 資訊蒐集

同時啟動 2 個 Agent（**單一訊息並行**）：

**Agent 1（外部資訊蒐集大師）prompt：**
```
你是「外部資訊蒐集大師」。任務：針對以下問題搜索外部資料。
1. 用 WebSearch 搜索 2-3 個不同角度的關鍵字
2. 用 WebFetch 讀取最相關的 2-3 個結果頁面
3. 彙整為結構化報告

輸出格式：
## 外部資訊報告
### 已知問題
### 技術方案（列出 2-3 個）
### 最佳實踐
### 關鍵發現

問題：{問題描述}
已嘗試的修正：{第 1 次修正內容與結果}
錯誤訊息/症狀：{具體錯誤}
```

**Agent 4（當前專案大師）prompt：**
```
你是「當前專案大師」。任務：在專案內搜集與以下問題相關的所有脈絡。
1. Glob/Grep 找到相關程式碼（函數定義、呼叫端、型別定義）
2. 讀取 _AIDocs/ 相關文件（先讀 _INDEX.md 定位）
3. git log --oneline -20 查相關檔案近期修改歷史
4. git blame 查問題行的來源

輸出格式：
## 專案脈絡報告
### 相關程式碼（附檔案路徑 + 行號）
### 歷史決策（git log/blame 結果）
### 架構脈絡（_AIDocs 摘要）
### 關鍵發現

問題：{問題描述}
涉及檔案：{相關檔案路徑}
```

主 AI 等兩者完成後，判斷資訊是否充足。不足則追加搜索。

---

### Phase 2 — 第 1 輪辯論

同時啟動 3 個 Agent（**單一訊息並行**），將 Phase 1 結果作為 context：

**Agent 2（正向策略大師）prompt：**
```
你是「正向策略大師」。基於以下資訊，分析問題根因並提出改進方案。

輸出格式：
## 正向策略報告
### 根因分析（為什麼第 1 次修正失敗）
### 修正方案（具體步驟，含程式碼片段）
### 預期效果
### 信心程度（高/中/低 + 理由）

{Phase 1 兩份報告全文}
```

**Agent 3（反向策略大師）prompt：**
```
你是「反向策略大師」。基於以下資訊，找出可能方案的盲點和風險。

輸出格式：
## 反向策略報告
### 每個方案的潛在問題
### 替代路線（正方沒考慮到的方向）
### 最壞情況分析
### 建議避開的陷阱

{Phase 1 兩份報告全文}
```

**Agent 5（落地執行大師）prompt：**
```
你是「落地執行大師」。基於以下資訊，評估每個方案的可行性。

輸出格式：
## 可行性報告
### 方案評估（每個方案：可行/有風險/不可行 + 原因）
### 優先順序建議（排序 + 理由）
### 落地障礙（需要的前置條件、可能的副作用）

禁止：不得給出數值化「命中率」，只做定性分析。杜絕幻想方案。

{Phase 1 兩份報告全文}
```

主 AI 綜合 3 者結果，識別共識與分歧。不確定處向使用者確認。

---

### Phase 3 — 第 2 輪深度挑戰

同時啟動 3 個 Agent（**單一訊息並行**），將 Phase 2 結果 + 分歧點作為 context：

**Agent 2（正向策略大師·第 2 輪）prompt：**
```
你是「正向策略大師」（第 2 輪深度挑戰）。
針對以下分歧點做深度分析。找出反方觀點的漏洞，或承認反方的合理性並調整方案。

輸出：調整後的最終建議方案 + 對反方質疑的逐條回應。

{Phase 2 三份報告全文 + 主 AI 識別的分歧點}
```

**Agent 3（反向策略大師·第 2 輪）prompt：**
```
你是「反向策略大師」（第 2 輪深度挑戰）。
針對以下分歧點做深度分析。找出正方調整後方案的殘留風險，或承認正方的合理性。

輸出：最終風險評估 + 對正方回應的逐條審視。

{Phase 2 三份報告全文 + 主 AI 識別的分歧點}
```

**Agent 5（落地執行大師·第 2 輪）prompt：**
```
你是「落地執行大師」（第 2 輪）。基於 2 輪辯論更新可行性評估。

輸出：最終優先建議（唯一推薦方案 + 備選方案）+ 執行步驟清單。

{Phase 2 三份報告全文 + 主 AI 識別的分歧點}
```

---

### Phase 4 — 最終決策與執行

1. 主 AI 綜合 2 輪結論，制定明確修正方案
2. 向使用者簡報方案要點，確認無異議後執行
3. 執行修正
4. 啟動 **Agent 6（垃圾回收大師）**：
```
你是「垃圾回收大師」。掃描以下修正產生的變動。

檢查清單：
1. 殘留的 debug 程式碼（console.log, print, debugger 等）
2. 過度修改（超出問題範圍的改動）
3. 暫存檔案（.bak, .tmp, 未清理的測試檔）
4. import 殘留（新增但未使用的 import）
5. 註解殘留（TODO, FIXME, HACK 等臨時註解）

輸出：需要清理的項目清單 + 建議動作。

修正涉及檔案：{修改的檔案清單}
Git diff 摘要：{git diff --stat 結果}
```

---

### Phase 5 — 自我驗證

1. 驗證修正是否真正解決問題（編譯/測試/手動確認）
2. **成功** → 向使用者回報：「精確修正計畫成效 ✓ — 問題已解決」
3. **失敗** → 向使用者說明：卡點 + 已嘗試方案，詢問是否調整方向
4. **連續 3 次**精確修正計畫未解決 → 強制暫停，與使用者全面對齊
